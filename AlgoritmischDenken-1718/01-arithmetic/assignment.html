Met behulp van lokale variabelen kan men de code wat leesbaarder maken:

  • We updaten toBePaidBack: we hebben coinsOf5Needed €5 munten reeds al wisselgeld, we moeten dus 5 × coinsOf5Needed minder terugbetalen.

  • We tellen het totaal aantal munten wisselgeld en bewaren het in totalCoinsNeeded. Dit geven we ook terug als eindresultaat van de berekening.

Door descriptieve namen te geven aan de lokale variabelen kan je beter begrijpen wat de berekeningen voorstellen. Het debuggen wordt ook veel eenvoudiger: je kan stap voor stap zien welke waarden toegekend worden aan elke lokale variabele.

Een vaak voorkomend patroon bij het wijzigen van een variabele is x = x + y, x = x - y, x = x * y, etc. M.a.w. de nieuwe waarde van de variabele hangt af van de oude waarde waarop een operator wordt toegepast. Hiervoor zijn verkorte notaties beschikbaar:

x = x + 1 x++

x = x - 1 x--
x = x * y x *= y
x = x / y x /= y

var vs let vs const

Er zijn meerdere manieren om een lokale variabele te introduceren. Hierboven gebruikten we telkens let, wat in feite nooit verkeerd is. Je bent dus vrij altijd let te gebruiken.

var is een verouderde manier om lokale variabelen in te voeren en kan in sommige omstandigheden tot heel verwarrende resultaten leiden. Het is aangeraden om in nieuwere versies van JavaScript let te gebruiken i.p.v. var.

const kan gebruikt worden indien je duidelijk wil maken dat een variabele zijn initiële waarde zal behouden, m.a.w. zal later niet gewijzigd worden.

advanceHour(h, n)

Een 24-urenklok geeft aan dat het nu h uur is (we negeren de minuten.) De klok heeft een knop waarmee je de tijd kunt aanpassen: elke duw op de knop verhoogt het uur met 1. Schrijf een functie advanceHour(h, n) die aangeeft welk uur de klok zal aangeven nadat je n keer geduwd hebt op de knop. Bijvoorbeeld, als de klok 15 aangeeft en je drukt 3 maal, is het resultaat advanceHour(15, 3) = 18.

De moeilijkheid ligt bij wat er moet gebeuren als je voorbij 23 uur gaat. Als h = 23 en n = 4, dan moet het resultaat 3 zijn.
